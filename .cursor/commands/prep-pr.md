변경된 파일들을 PR 올리기 전 점검해줘.

## 수행 순서

1. `git status`와 `git diff`로 변경된 파일을 확인한다.
2. `npm test`를 실행하고 결과를 요약한다.
3. 테스트가 통과하면 변경 사항을 **한국어 3줄**로 요약한다.
   - 테스트가 실패하면 3줄 요약 대신 실패 원인과 수정 방향을 안내한다.
4. Conventional Commits 형식(`feat:`, `fix:`, `chore:`, `refactor:`)으로
   **커밋 메시지 제목 1줄 + 본문**을 제안한다.
5. 마지막에 **리뷰어가 특히 봐야 할 부분**을 짚어준다.

## 제약

- 커밋, 푸시, PR 생성은 하지 않는다. 점검 결과만 보고한다.
- 코드는 수정하지 않는다.

## 출력 형식

### 1. 테스트 결과

- 통과/실패 여부
- 테스트 수, 실패 수, 소요 시간

### 2. 변경 사항 요약 (3줄)

- 한 줄당 하나의 핵심 변경

### 3. 커밋 메시지 제안

```
<type>: <제목>

<본문 — 변경 이유와 영향>
```

### 4. 리뷰 포인트

- 리뷰어가 집중해서 확인해야 할 파일·로직·엣지 케이스
